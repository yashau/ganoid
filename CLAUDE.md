# Ganoid — Claude Context

## Project overview

Ganoid is a Windows tool for managing multiple Tailscale coordination server profiles. It allows switching between self-hosted Headscale servers and the official Tailscale control plane without manual reconfiguration.

Two binaries:
- **`ganoidd`** — privileged Windows service. Manages Tailscale state, serves REST API + embedded web UI on `localhost:57400` (default).
- **`ganoid`** — system tray app. Polls `ganoidd`, shows status, triggers switches.

They communicate over local HTTP with a per-session bearer token. Either can start first — `ganoid` polls until `ganoidd` is available.

## Build process

**Always use `build.ps1`** — it handles versioninfo.json regeneration, goversioninfo, UI build, and Go compilation together:

```powershell
.\build.ps1 -Version 0.2.0
.\build.ps1 -Version 0.2.0 -Target all   # windows + linux + darwin
```

**Do not** manually run `goversioninfo` and then `build.ps1` — the script regenerates versioninfo.json itself. Running goversioninfo beforehand is redundant.

`make` is not available in the bash environment here — use PowerShell or `go build` directly.

`versioninfo.json` and `resource.syso` are in `.gitignore`. They are regenerated by `build.ps1` each time.

### Build from bash (quick, no UI rebuild)
```bash
GOOS=windows GOARCH=amd64 go build -ldflags="-s -w -H=windowsgui" -o ganoidd.exe ./cmd/ganoidd
GOOS=windows GOARCH=amd64 go build -ldflags="-s -w -H=windowsgui" -o ganoid.exe  ./cmd/ganoid
```

## Profile switching — the 8-step sequence

Implemented in `internal/manager/manager.go` `SwitchProfile()`.

**Critical order** (do not change without understanding the implications):

1. **Stop service** — must happen first so state files aren't locked/being written
2. **Verify + backup current state** — after stop, before any changes:
   - `stateFileReadable()`: checks `server-state.conf` is valid JSON (not corrupt); a fresh/empty state (official Tailscale with no profile data) is valid
   - `stateControlURL()`: if profile data exists, verify its ControlURL matches `currentProfile.LoginServer` — prevents backing up wrong profile's state
   - `rotateBackup()`: shifts existing backups `.v1`→`.v2`→`.v3`, drops `.v3`
   - `copyDir()`: backs up `%ProgramData%\Tailscale` → `%APPDATA%\ganoid\states\{profileID}`
3. **Clear** `%ProgramData%\Tailscale`
4. **Restore target state** — tries current backup then `.v1`, `.v2`, `.v3` in order:
   - `verifyState()`: must have at least one non-null `profile-XXXX` key
   - `stateControlURL()`: ControlURL must match `targetProfile.LoginServer`
   - If no valid version found: starts fresh (OK for first-time use of a profile)
5. **Write registry** — `HKLM\SOFTWARE\Tailscale IPN\LoginServer` = target URL (or delete key for official)
6. **Start service**
7. Finalize (placeholder send to keep total=8)
8. **Update active profile** in Ganoid config

**Why no logout step?** Logout was removed because it wipes profile data from `server-state.conf` before the backup runs. The entire state is being replaced anyway — logout is redundant.

**Why registry AND state files?** State files contain embedded `ControlURL` and are the primary source. The registry is a fallback for fresh profiles with no saved state — Tailscale reads it on first run. Registry is also synced at startup during reconciliation.

## Tailscale state file format

`%ProgramData%\Tailscale\server-state.conf` is a JSON file where values are base64-encoded:

```json
{
  "_current/S-1-5-...": "<base64: profile key name>",
  "_machinekey": "<base64: machine private key>",
  "_profiles": "<base64: JSON map of profile metadata including ControlURL>",
  "profile-XXXX": "<base64: full prefs JSON including ControlURL, PrivateNodeKey, etc>",
  "profile-YYYY": null
}
```

Key facts:
- `_profiles` base64-decodes to `{}` when a profile is cleared (e.g. after logout) — this is the "corrupted" state
- `profile-XXXX: null` means profile exists in index but has no data
- `ControlURL` is embedded in both `_profiles` and each `profile-XXXX` value
- `PrivateNodeKey` in `profile-XXXX.Config` is the actual auth credential — restoring this avoids re-authentication
- A **fresh/clean official Tailscale state** has all profiles null and `_profiles: {}` — this is valid, not corrupt

To decode a value: `[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($value))`

## Reconciliation (startup)

On every `ganoidd` startup, before serving (`cmd/ganoidd/main.go`):

1. Calls `tailscale debug prefs` to get actual `ControlURL`
2. Compares with `activeProfile.LoginServer`
3. If mismatch: searches profiles for matching URL; if found, sets it active; if not and URL is non-empty, creates a new auto-profile
4. Always syncs registry `LoginServer` key to match actual ControlURL

This handles cases where Tailscale's state was changed outside Ganoid (e.g. manual switch, service restart after failed switch).

## Logging

Package: `internal/logger`

- Levels: Debug, Info (default), Warn, Error
- Output: `%ProgramData%\Ganoid\ganoidd.log` AND stdout (file listed first in `io.MultiWriter` — Windows service has invalid stdout; if stdout is first, it short-circuits and nothing writes to file)
- Configured via `--log-level` flag on the service binPath
- Changed at runtime without restart via `set-log-level.cmd debug`

**Every action must have a log entry at debug level.** No exceptions. Every step start, result, and error.

`set-log-level.ps1` works by:
1. Reading current `sc.exe qc ganoidd` binPath
2. Stripping existing `-log-level` flag
3. Appending new `-log-level` value
4. Calling `sc.exe config ganoidd binPath= ...`
5. Restarting the service

## SPA routing

Two parts must both be correct:

1. **`ui/src/routes/+layout.js`**: `export const ssr = false` — makes `index.html` an empty shell (no prerendered content), so any route can load it and the client-side router takes over
2. **`internal/api/api.go` SPA fallback**: tries to serve the requested file; if not found, serves `index.html` using `http.ServeContent` (NOT `http.FileServer` — FileServer auto-redirects `/index.html` → `/` causing an infinite redirect loop)

`internal/tray/tray_windows.go` — pure Win32 via `golang.org/x/sys/windows`:

- `TrackPopupMenu` with `TPM_RETURNCMD` to get the selected item ID back directly
- Double-click opens browser

## API endpoints

All require `Authorization: Bearer {token}` header. Token is in `daemon.json`.

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/status` | Daemon version, active profile, Tailscale backend state + peer count |
| GET | `/api/profiles` | List profiles + active profile ID |
| POST | `/api/profiles` | Create profile |
| PUT | `/api/profiles/{id}` | Update profile |
| DELETE | `/api/profiles/{id}` | Delete profile (403 if active) |
| POST | `/api/profiles/{id}/switch` | SSE stream — triggers switch, streams 8-step progress |
| GET | `/api/tailscale/status` | Raw `tailscale status --json` output |

Static files served under the path prefix (from embedded `ui/dist`). Unknown paths fall back to `index.html` for SPA routing.

## Platform abstraction

`internal/platform/platform.go` defines the `Platform` interface. Only `windows.go` is fully implemented. `linux.go` and `darwin.go` are stubs that compile but return errors or empty values.

Windows-specific paths:
- Tailscale state: `%ProgramData%\Tailscale`
- Profile backups: `%APPDATA%\ganoid\states\{profileID}` (and `.v1`, `.v2`, `.v3`)
- Ganoid config/logs: `%ProgramData%\Ganoid`
- daemon.json: `%ProgramData%\Ganoid\daemon.json`
- Registry: `HKLM\SOFTWARE\Tailscale IPN\LoginServer`
- Tailscale binary: `%ProgramFiles%\Tailscale\tailscale.exe`

## Key gotchas

- **`tailscale set --login-server`** does not exist in Tailscale 1.96. Use registry + state restore instead.
- **`tailscale up --login-server`** prompts for re-authentication. Don't use it in the switch sequence.
- **`io.MultiWriter` short-circuits** on first error. Windows service has invalid stdout. Always put file writer first: `io.MultiWriter(file, os.Stdout)`.
- **`http.FileServer` redirects `/index.html` → `/`**. If your fallback serves `index.html` through FileServer, the handler catches `/` as a directory → serves index.html → FileServer redirects → infinite loop. Use `http.ServeContent` directly.
- **Tailscale state files override registry**. The `server-state.conf` embeds `ControlURL` and takes precedence over the registry on any non-fresh state. Registry is only authoritative for fresh (no state) starts.
- **`versioninfo.json` / `resource.syso` are gitignored**. They are regenerated by `build.ps1`. Do not manually commit them.
- **Tailscale service name** on Windows is `"Tailscale"` (the service, not the process).
- **`$GanoidPreview` must be read before elevation check** in install.ps1 — the variable doesn't survive UAC re-launch. It's preserved by passing `-Preview` as a CLI param to the temp script file.
- **After installing a new version**, the service `binPath` is reset so any `-log-level debug` flag is lost. Run `set-log-level.cmd debug` again.
